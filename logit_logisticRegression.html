3. Dealing with Ordinal IVs in Logit Models

Notice how the gdp.percap.5cat variable is repeated and has weird suffixes? R is trying to fit the ordered factor variable to different polynomial functions to see which one makes the most sense.

  • .L = Linear model (treats the effect of the IV as usual, like slope * Xâ€¦)
  • .Q = Quadratic model (models the IV as slope * X^2)
  • .C = Cubic model (models the IV as slope * X^3)
  • ^4 = Quartic model (models the IV as slope * x^4)
  • â€¦and so on for as many categories as there are in the variable, minus 1â€¦

This happens because categorical variables (even ordinal ones) donâ€™t necessarily have a linear or consistent effect on the DV at different levels. R tries to find the best â€śshapeâ€ť of the IV-DV relationship

In our case, because world$gdp.percap.5cat.L is the only significant one of the 4 options, itâ€™s likely that the effect of GDP per capita on the probability of being a democracy is fairly linear and consistent.

But thereâ€™s a less math-y way to understand the relationship: When we treat it like itâ€™s unordered (i.e., a nominal variable), R just converts it to a set of dummies, which we can interpret as usual.

We can do that using the factor() function with the unordered = T option directly in the modelâ€¦

7. Advanced Material

Manually Calculating Model Stats for glm() Models

Weâ€™ll start by running our fully specified model (logit1 just like before), but then weâ€™ll run another model: the null model (logit0 in this case), which is the same DV but without any of the IVs to help predict it. The null model acts as a baseline for comparison to our model.

logit1 <- glm(states$deathpen.status ~ states$south + states$attend.pct + states$public.conservative + states$gun.dealers, family = binomial)

logit0 <- glm(states$deathpen.status ~ 1, family = binomial)

Then, to get the p-value of the model as whole, we have to compare the log likelihood of our model to the log likelihood of the null model using a likelihood ratio test:

pvalue <- anova(logit0, logit1, test = "Chisq")

Wait. ANOVA? Chi-square? Again??

Yep.

Earlier on, when we looked at bivariate tests, ANOVA helped us compare means and Chi-square helped us analyze crosstabs with categorical variables. With logistic regression, weâ€™re asking different questions:

  • Does a model with IVs fit the data significantly better than a model without IVs?

So, R is just using the anova() function to compare the log likelihoods of both those models to produce a likelihood ratio test statistic, then using a Chi-square test to determine whether the improvement in model fit is large enough to be significant.

Next, we can calculate McFaddenâ€™s pseudo R-squared to see how much the full model improves the log-likelihood compare to the null model. So, we specify the logLik() parameters stored in each model:

pseudoR2 <- as.numeric(1 - (logLik(logit1) / logLik(logit0)))

Similarly, we can extract the coefficient parameters from the stored model, so we can exponentiate them into Odds Ratios:

oddsRatios <- exp(coef(logit1))
