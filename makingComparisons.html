2. Base R table() Function

Base R is what R comes with a fresh install, meaning it has no extra packages. The table() function in base R creates a table from a variable or set of variables, so it can help us get a quick sense of the distribution of observations across categories.

We can look at variables individually….

table(states$abortlaws.3cat)

     Few restrictions Moderate restrictions     Many restrictions 
                   11                    20                    19 
table(states$religiosity3)

 Low  Mid High 
  17   17   16

…or view at them together to see how the categories of one variable align with the categories on another one. In other words, we’re looking at how many cases fit in each box of the table to understand how the two variables are related (if at all).

table(states$abortlaws.3cat, states$religiosity3)
                       
                        Low Mid High
  Few restrictions       10   1    0
  Moderate restrictions   7   9    4
  Many restrictions       0   7   12

But what about our column percentages???

This minimal code works, but how does R know which one is the DV and which one is the IV? The default order of the arguments matters. The DV always comes first, then the IV. Oof. Multiple long value labels are a nightmare; they make both the table and the chart difficult to read.

But the function includes optional argument to change the labels! The dvlabs = c() is for categories of the DV and ivlabs = c() is for the categories of the IV.

Because we’re more interested in column percentages than the raw counts, too, we also include the compact = T argument to make the output clearer.

Try the code below (it spans multiple lines just to help delineate each option):

crosstabC(dv = nes$libcon3, iv = nes$educ.5cat, 
          dvlabs = c("Lib", "Mod", "Con"), 
          ivlabs = c("<HS", "HS", "SomeBA", "BA", ">BA"), 
          plot = "bar", 
          plot.response = "Lib", 
          compact=T
          )

5. Adding Complexity to the Crosstabs

Instead of specifying only “Lib” as the response category of the DV that we want to show, we have the option to do all of the categories at once in a clustered bar chart: plot.response = "all".

crosstabC(dv = nes$libcon3, iv = nes$educ.5cat, 
          dvlabs = c("Lib", "Mod", "Con"),
          ivlabs = c("<HS", "HS", "SomeBA", "BA", ">BA"), 
          plot = "bar", 
          plot.response = "all", # all the DV categories
          compact=T
          )
===========================================================================
                         Cross-Tabulation Analysis

% Con     24.59    33.09     30.92     25.34     20.26     27.14
Count    244.00   970.00   2322.00   1898.00   1525.00   6959.00

By changing plot.response from a specific category (e.g., “Lib”) to “all”, we get the clustered bar chart! Plus, we can still adjust the chart type with the plot = option is we need a stacked line graph or a mosaic plot.

6. Comparing Continuous/Interval DVs

Comparing the mean of a continuous/interval DV across different categories of a categorical IV is often useful. Here’s one from the Pollock textbook:

compmeansC(dv=legalclimate, iv=judge.selection, data=states)
===========================================================================

===========================================================================


Table: Mean Values of legalclimate by judge.selection

                             Mean    N   St. Dev.
-------------------------  ------  ---  ---------
Appointed by commission     66.81   21       5.12
Appointed by governor       63.14    5       8.55
Appointed by legislature    63.85    2       6.29
Non-partisan election       62.02   15       5.96
Partisan election           56.13    7       7.91
Total                       63.39   50       6.96
One or more text labels for bars is long. Consider abbreviating with ivlabs argument.

Oh. Well, we could use the ivlabs = c() option again in this function. But let’s just try to compare the mean GDP per capita across different regime types. Polity score is a pretty common measure of the level of democracy in a country:

compmeansC(dv=world$gdp.percap, iv=world$polity.score, plot = "line")
Error in compmeansC(dv = world$gdp.percap, iv = world$polity.score, plot = "line"): There's a problem: world$polity.score is numeric variable. Collapse values to fit a mean comparison table.

WTF? Turns out polity.score is technically ordinal, but it’s “too numeric” because it has a lot of categories (-10 to +10). Since the numbers don’t have labels and it’s just seen as an interval variable, R is confused about how to treat it discretely/categorically.

Let’s try a more clearly categorical measure of democracy like vdem.4cat instead:

compmeansC(dv=world$gdp.percap, iv=world$vdem.4cat, plot = "line")

Success! Sort of…

We still have a graph problem. Time to re-label!

compmeansC(dv=world$gdp.percap,

The labels are definitely easier to read, but the scientific notation on the y-axis is confusing. So, let’s do a quick transformation of the DV (world$gdp.percap) to change the scale first, then use our new variable instead.

gdppc.thou <- world$gdp.percap/1000 #GDP per capita in thousands of dollars

compmeansC(dv=gdppc.thou, 
           iv=world$vdem.4cat, 
           plot = "line", 
           ivlabs = c("LibDem", "Dem-ish", "Autoc-ish", "FullAutoc")
           )

----------  ------  ----  ---------
LibDem       47.78    31      20.10
Dem-ish      15.90    55      11.30
Autoc-ish    10.18    59      14.09
FullAutoc    19.71    23      23.90
Total        20.30   168      21.00

Slay. Or whatever.

Now, let’s add confidence intervals (plot.ci = T) to the code, so we can see if there is a significant difference between the means of gdppc.thou across regime types,

compmeansC(dv=gdppc.thou, 
           iv=world$vdem.4cat, 
           plot = "line", 
           plot.ci=T, 
           ivlabs = c("LibDem", "Dem-ish", "Autoc-ish", "FullAutoc")
           )
===========================================================================
                         Mean Comparison Analysis
===========================================================================


Table: Mean Values of gdppc.thou by world$vdem.4cat

              Mean     N   St. Dev.
----------  ------  ----  ---------
LibDem       47.78    31      20.10
Dem-ish      15.90    55      11.30
Autoc-ish    10.18    59      14.09
FullAutoc    19.71    23      23.90
Total        20.30   168      21.00

Understanding Confidence Interval bars:

  • If the CI bars touch/overlap: Not significant.
  • If they don’t: Significant.

In our case, there is a significant difference in the mean of gdppc.thou for “LibDem” countries relative to “Dem-ish” and “Autoc-ish” countries. But there’s some overlap comparing the other regime types.

7. Boxplots with Interval/Continuous DVs Across Categorical IVs

We already know that the mean is affected by skewness, so we should check the distribution of gdppc.thou to make sure that things aren’t too wonky…

We can do that with a boxplot:

boxplotC(dv=gdppc.thou, 
         iv=world$vdem.4cat, 
         ivlabs = c("LibDem", "Dem-ish", "Autoc-ish", "FullAutoc")
         )
================================================================================
                       Making Comparisons with Box Plots
================================================================================

Here’s how to read a boxplot:

  • dots: outliers
  • top line: maximum, excluding outliers
  • bottom line: minimum, excluding outliers
  • top of the box: third quartile (75% of data under this line)
  • bold line: median
  • bottom of the box: first quartile (25% of data under this line)

8. Scatterplots with Interval/Continuous DVs and IVs

When we have continuous/interval variables in both the IV and the DV, we can do a quick scatterplot.

But the plot() function expects the IV (x =) first by default, which throws me off because I’m used to the DV coming first in the code! So, that’s another reason to get in the habit of specifying which is which in the code itself.

plot(y=world$gdp.percap, x=world$pop.urban)

Yikes, this is a messy scatterplot.

Transforming the skewed gdp.percap variable using its natural log reduces the skew and clarifies the pattern in the scatterplot.

plot(y=log10(world$gdp.percap), x=world$pop.urban)

Now, let’s make the chart clear to the consumers of our visualization by adding titles (main =) and axis labels (xlab = and ylab =)…

plot(y=log10(world$gdp.percap), x=world$pop.urban, 
     main = "Scatterplot of Urban Pop and Wealth", 
     xlab = "%Urban Population", 
     ylab = "Per Capita GDP (natural log)"
     )
